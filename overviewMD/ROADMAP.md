# Roadmap

## Phase 0: Foundation
- finalize product language
- finalize service boundaries
- draft object model
- establish Docker-first development setup
- decide repo structure

## Phase 1: Core workspace
- company creation
- project creation
- company/project switcher
- project brief storage
- Hermes session mapping
- active session list
- session detail view

## Phase 2: Guidance layer
- guidance events
- subtle nudges
- summary requests
- blocker prompts
- reorientation prompts
- visible guidance history in session timeline

## Phase 3: Reports and visibility
- session summary reports
- project overview reports
- executive briefs
- recent activity timeline
- delegation visibility

## Phase 4: Analytics
- session activity metrics
- delegation counts
- cost/token trends
- stalled session signals
- project-level performance snapshots

## Phase 5: Automation and refinement
- scheduled reporting
- scheduled nudges
- richer hooks integration
- optional worker service
- reusable project templates
- permissions and collaboration upgrades

## Milestone definition for first usable build

The first usable build should let a person:
- create a company
- create a project
- open a project workspace
- start or resume a Hermes session
- send a guidance event
- review a report
- inspect basic analytics
